## Phase 1: Update V3 linked harmonizer to match V2

The V3 linked file is used for IMR research — the exact use case where the LBW-IMR divergence bugs were discovered. It must have the same birth-side improvements as V2.

### 1a. Update `scripts/03_harmonize/harmonize_linked_v3.py`

The V3 linked harmonizer has its own `OUT_SCHEMA` and its own per-era processing logic (similar structure to `harmonize_v1_core.py`). It needs these birth-side additions:

**Add to `OUT_SCHEMA`** (in the same positions as V2's schema — after `marital_status` and after `maternal_race_detail`):
- `("marital_reporting_flag", pa.bool_())` — after `marital_status`
- `("race_bridge_method", pa.string())` — after `maternal_race_detail`

**Add 6 clinical detail columns to `OUT_SCHEMA`** (these may already be partially wired; check):
- `("pre_pregnancy_diabetes", pa.bool_())`
- `("gestational_diabetes", pa.bool_())`
- `("nicu_admission", pa.bool_())`
- `("weight_gain_pounds", pa.int16())`
- `("induction_of_labor", pa.bool_())`
- `("breastfed_at_discharge", pa.bool_())`

**Add extraction logic** (mirror the V2 harmonizer's approach):
- `marital_reporting_flag`: extract from `F_MAR_P` for 2014+ batches (1→True, 0→False, blank→null); null for pre-2014.
- `race_bridge_method`: year-based — `"nchs_bridged"` for years < 2020, `"approximate_from_detail"` for years >= 2020.
- **2020+ race reconstruction**: for non-Hispanic births where `race_bridged` (MBRACE) is null, reconstruct `maternal_race_ethnicity_5` from MRACE6 detail codes (01→NH_white, 02→NH_black, 03→NH_aian, 04/05→NH_asian_pi, 06→null).
- Clinical detail columns: extract `RF_PDIAB`, `RF_GDIAB`, `AB_NICU`, `WTGAIN`, `LD_INDL`, `BFED` using `_yn_to_bool()` or `_to_int_or_null()` as appropriate. Null for pre-2014 years.

**Add to the output table assembly** in the same column order as V2.

**Reference implementation**: `scripts/03_harmonize/harmonize_v1_core.py` — search for `marital_rpt_flag`, `race_bridge`, and the clinical detail extractions to see exactly how they're done in V2. The linked harmonizer should mirror this logic.

**Key difference**: The linked harmonizer uses `LINKED_BIRTH_2014_2020_FIELDS` for field specs (which inherits from `PUBLIC_US_2014_2015_FIELDS` with a birthweight position override). `F_MAR_P` is at position 123 in both specs, so it's already being extracted at the import stage.

### 1b. Update `scripts/04_derive/derive_linked_v3.py`

Add the 3 medical risk factor booleans to the derived columns:

```python
pa.field("diabetes_any_bool", pa.bool_()),
pa.field("hypertension_chronic_bool", pa.bool_()),
pa.field("hypertension_gestational_bool", pa.bool_()),
```

Add the derivation logic (same as V2's `derive_v1_core.py`):

```python
def _int129_to_bool(arr):
    return pc.if_else(
        pc.equal(arr, pa.scalar(1, type=arr.type)),
        pa.scalar(True),
        pc.if_else(
            pc.equal(arr, pa.scalar(2, type=arr.type)),
            pa.scalar(False),
            pa.scalar(None, type=pa.bool_()),
        ),
    )
```

Apply to `diabetes_any`, `hypertension_chronic`, `hypertension_gestational` columns from each batch.

## Phase 8: Upload to Zenodo

### Option A: GitHub integration (recommended if repo is on GitHub)

1. Go to https://zenodo.org and log in (create account if needed)
2. Go to https://zenodo.org/account/settings/github/
3. Enable the `yoelplutchok/natality-harmonization` repository
4. Go to GitHub and create a Release from the `v2.5.0` tag
5. Zenodo will automatically create a deposit and mint a DOI
6. Edit the Zenodo deposit to fill in any missing metadata

**Limitation**: This uploads the full repo contents (code + docs), but NOT the large Parquet files (they're gitignored). You'll need to manually upload the Parquet files to the deposit.

### Option B: Manual upload (recommended for large data files)

1. Go to https://zenodo.org/deposit/new
2. Fill in metadata from `.zenodo.json` (or it may auto-populate if you have the GitHub integration)
3. Upload files. **Upload these files and only these files:**


4. Set the **Upload type** to "Dataset"
5. Set the **License** to "Creative Commons Attribution 4.0 International"
6. Fill in **Title**: "Harmonized U.S. Natality and Linked Infant Death Microdata, 1990-2024"
7. Fill in **Authors**: Your name (and ORCID if available)
